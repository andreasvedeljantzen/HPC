Intel® Processor Graphics Samples

The compiler samples included with the Intel® C++ Compiler with Intel® Processor Graphics offload are designed to illustrate specific compiler features, programming concepts, recommended coding patterns, and compiler optimizations. All examples are based on the unified running and measurement framework. Each example resides in a separate directory and may include several implementation variants.

Quick Start on Windows*
  1. Start Visual Studio and open the solution file matching the Visual Studio version used, e.g. processor_graphics_samples\processor_graphics_samples_2013.sln.
  2. Compile the project.
Quick Start on Linux*
  1. Compile the project using the provided Makefile.
System Requirements:
    You will need:
    • A processor that supports Intel® Graphics Technology
    • A driver that supports it
    • For Windows*, a copy of binutils with your PATH set to include the directory containing ld.exe
    For details on what is required, please refer to Getting Started With Compute Offload to Intel® Graphics Technology.
Build Instructions:
Refer to the individual readme.html file included with each individual sample and tutorial for additional information about building and executing the example.
For Microsoft Visual Studio* users:
    The samples package contains Visual Studio solution files named processor_graphics_samples_201x.sln configured to use the Intel® C++ Compiler. The solution promotes common use cases for Visual Studio projects such as building, running, or debugging an application. Below are the instructions for key tasks which can be done with the samples.

    To Compile All of the Projects in the Solution

    • Click Build > Rebuild Solution
For Microsoft Windows* Command Line users:
    The samples package also contains one command line script to build all the samples.

  • Enable your particular compiler environment for the Intel® C++ Compiler:

    • To Compile All of the Samples

    • Run build-all.bat
    • The resulting sample executables are placed in the out directory.
For Linux* users:
    The samples package contains a Makefile configured to use Intel® C++ Compiler. The Makefile promotes common use cases such as building, running, or debugging an application. Below are the instructions for key tasks which can be done with the samples.

  • Enable your particular compiler environment for the Intel® C++ Compiler:

    • To Build All of the Targets in the Makefile

    • Run make all

    To Run the Targets in the Makefile

    • Run make run

    To Debug

    • Run make all debug=1


      • To build the samples with debug information. To debug the application on the target please follow the instructions in the User's Guide on using the debugger for Intel® Graphics Technology.
      The application can also be debugged by running the application on the host only, e.g. convolution exec=offload_off. Debug the code by setting breakpoints in gdb* and then running the application on the host.
      • Set a breakpoint in the source code after the #pragma offload statement.
      • Run in the debugger to hit the breakpoint.
Project Descriptions
    Project Description

    Convolution

    Contains 2 offload implementations of 2D image convolution, with filter image size defined at runtime. Illustrates vector functions, explicit caching of data for fast access in local register arrays, and using Intel® CilkTM Plus Array Notations for compact description of data parallel operations.

    CrossFade

    Contains 2 offload implementations of a very simple filter merging 2 images.

    HistogramLocals

    Contains 2 offload implementation of byte histogram:

    • For a regular byte array (1 byte per pixel)

    • For array or 4-component pixels (4 bytes per pixel), separate for each components

    Illustrates calculation of the histogram of a byte array using local arrays in register to cache data blocks in temporary histograms.

    Matmult

    Contains 3 offload implementations of matrix multiplication. Supports either double or float using C++ templates. Instantiated workloads are:

    • Trivial: Matmult (float), Matmult_d (double)

    • Tiled (faster): MatmultLocalsAN (float), MatmultLocalsAN_d (double), MatmultLocals (float).

    Illustrates outer loop vectorization using Intel® CilkTM Plus pragma simd, explicit caching of data tiles for fast access in local register arrays, and using Intel® CilkTM Plus Array Notations for compact description of data parallel operations.

    NBody

    Contains 3 offload implementations of N body dynamics. Supports either double or float using C++ templates. Also illustrates a simple technique to share a task between CPU and GPU. Instantiated workloads are:

    • Trivial using vector function: NBody (float)

    • Optimized via caching in register arrays: NBodyLocals (float), NBodyLocals_SOA (float)

    TestGFXVector

    Contains several kernels illustrating use of GFXVector4<T> and GFXVector3<T> utility class enabling operating on 3 or 4 component structures in a compact way using overloaded operators (similarly to OpenCL* float4 type). Also illustrates a simple technique to share a task between CPU and GPU.

    MatmultSLM

    Contains kernel illustrating matrix multiplication and utilizing Shared Local Memory feature via the new programming model extensions.
