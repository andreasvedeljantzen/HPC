Reverse debugging with the GNU* Project Debugger for Intel® Architecture

Getting Started

For prerequisites and installation see the GNU* Project Debugger Release Notes.

Starting the Example in Eclipse*

  1. Start the preconfigured Eclipse* IDE from:

    <install-dir>/eclipse_<version>

    (For example: opt/intel/eclipse_mars)

    When you start Eclipse* with Intel® System Studio for the first time, the welcome screen appears, providing an overview of Intel® System Studio components and a list of links to further information. These links are also available in the Intel® System Studio Menu in the Eclipse* menu bar.

  2. Scroll to section Debuggers and Analyzers > Intel® Enhanced GDB Application Debugger and click Samples.

    The internal browser window opens the web page for the Application Debuggers samples inside Eclipse*.

  3. Go to GDB Reverse Debugging Sample and click Download Now.
  4. Click Import Project to import the demo project directly into your Eclipse* workspace.
  5. Alternatively you can save the project to your local system and import it into Eclipse* manually:
    1. Go to File > Import > General > Existing Projects Into Workspace.
    2. Browse for the demo project file GDB_ReverseDebuggingSample.tgz.
    3. Click Finish to import the demo project into your workspace.
  6. Select Window > Show View > Project Explorer to open the Project Explorer view.

Enabling Reverse Debugging Tool Bar Items

Select Window > Perspective > Customize Perspective....

Switch to the tab Action Set Availability and check Reverse Debugging in the Available Action Sets column.

Building and Launching the Application for Execution on the Host System (Native)

To build the application:

  1. Go to Project > Properties > C/C++ Build > Tool Chain Editor > Current toolchain and choose the Intel® System Studio toolchain.
  2. Select Project > Build Configurations > Set Active > 1 Debug and set the Debug configuration as the active configuration.
  3. Select Project > Build All to build the application.

    You can ignore warnings about unused functions.

To create a debug launch configuration:

  1. Select Run > Debug Configurations....

  2. Double-click C/C++ Application and select GDB_ReverseDebuggingSample Debug.

  3. Switch to the Debugger tab.
  4. Click Select Other to select the launcher GDB-IA (DSF) Create Process Launcher.

    The launcher automatically sets the provided GDB. The text field GDB debugger should list gdb-ia.

  5. On the tab Main of the Debugger Options, check Enable Reverse Debugging at startup.
  6. Select a recording method from the dropdown menu that is appropriate for your system. See the Release Notes for more information.
  7. Click Apply to save your debug launch configuration.

To launch the application:

  1. Select Run > Run Configurations > C/C++ Application > GDB_ReverseDebuggingSample Debug > Main.
  2. Click Debug.

Building and Launching the Application for Execution on the Target System (Remote)

Prerequisites:

To perform remote debugging on the target system, make sure you install and enable the remote debug agent, gdbserver, on the target. For more information refer to the getting started guide for Linux* host or to the Release Notes.

To build the application:

  1. Go to Project > Properties > C/C++ Build > Tool Chain Editor > Current toolchain and choose the Intel® System Studio toolchain.
  2. Select Project > Build Configurations > Set Active > 1 Debug and set the Debug configuration as the active configuration.
  3. Select Project Properties > C/C++ Build > Settings and set the GNU_PREFIX and SYSROOT:
    • For Wind River* Linux*: set SYSROOT=<WRL6_path>/qemux86-64set GNU_PREFIX=x86_64-wrs-linux-
    • For Yocto Linux*: set SYSROOT=<YL_path>/sysroots/core2-32-poky-linuxset GNU_PREFIX=i586-poky-linux-
  4. Select Project Properties > C/C++ Build > Environment and update the path in the build environment:
    • Wind River* Linux*: Append the following to the PATH environment variable: <WRL6_path>/x86_64-linux/usr/bin/x86_64-wrs-linux;
    • Yocto Linux* append the following to the PATH environment variable: <YL_path>/sysroots/core2-32-poky-linux/i586-poky-linux;
  5. Select Project > Build All to build the application.

    You can ignore warnings about unused functions.

To create a debug launch configuration:

  1. Select Run > Debug Configurations....

  2. Double-click C/C++ Remote Application and select GDB_ReverseDebuggingSample Debug.

  3. The tab Main of the debug configuration should already contain the correct entries in the Project and C/C++ Application fields. If not, use the respective Browse buttons to browse for and fill in the appropriate information.

  4. Switch to the Debugger tab.
  5. Click Select Other to select the launcher GDB-IA (DSF) Manual Remote Debugging Launcher.

    The launcher automatically sets the provided GDB. The text field GDB debugger should list gdb-ia.

  6. Select a recording method from the dropdown menu that is appropriate for your system. See the Release Notes for more information.

  7. On the tab Connection of the Debugger Options, enter the target host name /IP address and port number to be used for the debug session. A loop back connection on a standard Linux* system is used for this tutorial, so the host name will be "localhost". Choose a port number that is not used by other services, such as 2345.

  8. To use a remote system running an OS that is supported by Intel® System Studio, for example Wind River* Linux or Yocto* Linux, switch to the tab Shared Libraries of the Debugger Options and click Add... to add the path to the target shared objects of the SYSROOT to the debug launch configuration. Note that the number of directories and the location that need to be added may vary for different target operating systems.
  9. Click Apply to save your debug launch configuration.

To launch the application:

  1. Select Run > Run Configurations > GDB_ReverseDebuggingSample Debug > C/C++ Remote Application > Main.
  2. Click Debug.
