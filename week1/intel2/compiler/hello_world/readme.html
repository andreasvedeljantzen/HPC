Intel速 C++ Compiler Hello World Sample

This is a simple "Hello World" example that illustrates how to test whether or not your code was compiled using Intel速 C++ Compiler. This sample will work from Eclipse* or from the Command Line.

System Requirements:
  • Hardware:
    • Any Intel® processor like 2nd Generation Intel® Core™ i3, i5, or i7 processors and Intel® Xeon® E3 or E5 processor family
  • For Linux*:
    • One of the following compiler toolchain environments:
      • GNU* GCC 4.5 or newer
      • Wind River Linux* 6, 7 or 8
      • Yocto Linux* 1.7, 1.8 or 2.1
    • Intel® System Studio 2017 Composer Edition for C++ Linux*
  • Abbreviations used:
    • <ISS_INTALL_PATH>: refers to the path Intel® System Studio is installed.
      • On Linux* System with default installation settings: /opt/intel/system_studio_2017.x.y
    • <WRL_path>: refers to the path where Wind River Linux* is installed.
    • <YL_path>: refers to the path where Yocto Linux* is installed.
      • On Linux* System with default installation settings for Intel® Edison: /opt/poky-edison/1.7/
    • <SYSROOT>: refers to the path where SYSROOT directory is located.
      • Usually <YL_path>/sysroots or <WRL_path>/sysroots

Build Instructions

For Eclipse usage with the Intel速 C++ Compiler:
  1. Open Eclipse* and import the sample into your workspace from Eclipse* menu: (File > Import > General > Existing Projects into Workspace )
  2. Make sure you are using the Intel® System Studio toolchain from (Project Properties > C/C++ Build > Tool Chain Editor > Current toolchain).
  3. Set the GNU_PREFIX and SYSROOT in the (Project Properties > C/C++ Build > Settings)
    • For Wind River* Linux* it would be
      set SYSROOT=<WRL7_path>/qemux86-64
      set GNU_PREFIX=x86_64-wrs-linux-
    • For Yocto Linux* it would be
      set SYSROOT=<YL_path>/sysroots/core2-32-poky-linux
      set GNU_PREFIX=i586-poky-linux-
  4. The path setting (Project Properties > C/C++ Build > Environment) in the build environment needs to be updated as well
    • Wind River* Linux* append the following to the PATH environment variable:
      <WRL_path>/x86_64-linux/usr/bin/x86_64-wrs-linux;
    • Yocto Linux* append the following to the PATH environment variable:
      <YL_path>/sysroots/core2-32-poky-linux/i586-poky-linux;
  5. Create a connection to the Target System
    • Go to Connection settings (Run > Run Configurations > C/C++ Remote Application > Main)
    • Create a new SSH Connection by pressing on the New button next to Connection
    • Specify the host and the connection name and hit on Finish
  6. Build the project. Use one of the following build configurations Release or Debug to build the different variants.
  7. Transfer the application on the Linux* based target by running the sample application
  8. You will notice the output "Intel(R) C Compiler was used!"
For Eclipse usage with a non-Intel速 Compiler:
  1. Change the toolchain used (Project Properties > C/C++ Build > Tool Chain Editor > Current toolchain) to Cross GCC.
  2. Make sure you have the correct compiler command settings.
  3. If you do not have a connection already, create a connection to the Target system
    • Go to Connection settings (Run > Run Configurations > C/C++ Remote Application > Main)
    • Create a new SSH Connection by pressing on the New button next to Connection
    • Specify the host and the connection name and hit on Finish
  4. Build the project. Use one of the following build configurations Release or Debug to build the different variants.
  5. Transfer the application on the Linux* based target by running the sample application
  6. You will notice the output "Intel(R) C Compiler was NOT used!"
For Command Line usage with the Intel速 C++ Compiler:
  1. Open a Terminal screen
  2. Configure the compiler variables.
    cd <ISS_INSTALL_PATH>/compiler_libraries/linux/bin/
    source compilervars.sh ia32
    export PATH=<SYSROOT>/x86_64-pokysdk-linux/usr/bin/i586-poky-linux/:$PATH
    export SYSROOT=<SYSROOT>/core2-32-poky-linux
    export GNU_PREFIX=i586-poky-linux-
  3. Change directory to your sample's path.
  4. Compile the sample using Intel® C Compiler.
    icc -gnu-prefix=$GNU_PREFIX --sysroot=$SYSROOT ./src/HelloWorld.c -o hello_world_intel
  5. Transfer the binary file hello_world_intel to the Linux* based target.
  6. Run the binary file on your target system ./hello_world_intel
    7. You might need to add the executable permission using chmod +x hello_world_intel
  7. You will notice the output "Intel(R) C Compiler was used!"
For Command Line usage with a non-Intel速 Compiler:
  1. Open a Terminal screen
  2. Change directory to your sample's path.
  3. Compile the sample using a NON-Intel® C Compiler.
  4. Transfer the binary file hello_world_nonIntel to the Linux* based target.
  5. Run the binary file on your target system ./hello_world_nonIntel
    6. You might need to add the executable permission using chmod +x hello_world_nonIntel
  6. You will notice the output "Intel(R) C Compiler was NOT used!"
